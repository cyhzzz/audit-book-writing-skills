# 项目概览 (Project Overview)

## 📌 项目简介

审计书籍智能校对优化技能（Audit Book Smart Proofreading）是专为审计书籍编写的智能校对优化工具。该项目基于真实的审计书籍章节校对优化场景，提炼出一套系统化、智能化的校对优化工作流。

### 项目背景

在审计书籍编写过程中，面临以下核心挑战：

1. **法规准确性问题** - 法规条文表述不准确、条款序号错误、时间标注混乱
2. **案例权威性问题** - 使用非权威案例、案例来源不明确、编造案例
3. **内容合规性问题** - 离题内容、超范围内容、重复内容
4. **结构清晰性问题** - "审计内容"与"审计重点"重复、逻辑冲突
5. **效率低下问题** - 人工校对耗时、容易遗漏、质量不稳定

为解决这些问题，本项目基于真实场景的 26 步校对优化流程，开发了智能校对优化技能。

### 核心模块

#### 1. 智能标记模块 (Smart Tagging Module)

**功能:**
- 自动识别文档中 8 种类型的内容
- 应用识别规则进行标记
- 生成标记报告

**识别规则:**
- 法规条文（FA1/FA2）
- 条款引用（FB1/FB2）
- 发文字号（FC1/FC2）
- 显式地名案例（A1）
- 隐式地名案例（A2）

#### 2. 多源法规验证模块 (Multi-Source Regulation Verification Module)

**功能:**
- 加载 1-8 个法规汇总文件
- 逐一比对法规条文
- 自动修正条款序号和文字表述
- 标记无法验证的内容

**验证流程:**
- 相似度匹配
- 条款序号修正
- 文字表述覆盖
- 时间标注标准化

#### 3. 权威案例替换模块 (Authoritative Case Replacement Module)

**功能:**
- 识别需要替换的案例
- 按优先级搜索官方网站
- 替换案例并标注来源
- 删除无替换案例的内容

**优先级:**
1. 国家文物局官方网站
2. 浙江省文物局官方网站
3. 其他各省文物局官方网站

#### 4. 内容智能扩充模块 (Intelligent Content Expansion Module)

**功能:**
- 筛选相关法规条文
- 查找权威案例
- 生成审计应用要点
- 按模板组织内容

**扩充结构:**
- 法规条文原文
- 相关案例
- 审计应用

#### 5. 结构优化模块 (Structure Optimization Module)

**功能:**
- 识别重复内容
- 识别逻辑冲突
- 重新分配内容
- 优化文档结构

**优化策略:**
- 基础性内容 → "审计内容"
- 关键性内容 → "审计重点"

#### 6. 质量检查模块 (Quality Check Module)

**功能:**
- 多维度质量检查
- 生成质量报告
- 标注待改进项

**检查维度:**
- 法规准确度
- 案例权威性
- 内容完整性
- 结构清晰度

```

│    - 基于法规补充条文原文                                      │
│    - 查找相关案例                                              │
│    - 生成审计应用要点                                          │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐

│    - 识别并归并重复内容                                        │

                            ↓
┌─────────────────────────────────────────────────────────────┐
```

### 1. 智能化程度高

- **详细报告** - 每个阶段生成详细报告，可追溯

### 4. 使用便捷

- **会话式交互** - 自然语言交互，易于使用
- **分阶段确认** - 每个阶段完成后确认，确保质量

## 🚀 应用场景

### 场景 1: 审计书籍章节校对优化

**需求:** 对审计书籍的某一章节进行校对优化，确保法规准确、案例权威。

**输入:**
- 待校对文档（如"古人类遗址保护.doc"）
- 法规汇总文件（1-8个）
- 审计主题（如"古人类遗址保护责任审计"）

**输出:**
- 校对后的最终文档
- 详细报告

### 场景 2: 法规类文档审核

**需求:** 对法规类文档进行审核，确保法规条文表述准确。

**输入:**
- 待审核文档
- 法规汇总文件

**输出:**
- 审核后的文档
- 审核报告

### 场景 3: 案例型文档标准化

**需求:** 对案例型文档进行标准化，替换为官方权威案例。

**输入:**
- 待标准化文档

**输出:**
- 标准化后的文档
- 替换报告

### 场景 4: 审计报告质量提升

**需求:** 对审计报告进行质量提升，优化文档结构和内容。

**输入:**
- 待提升报告
- 相关法规资料

**输出:**
- 提升后的报告
- 优化报告
