## 🎯 核心功能

### 1. 四轮评审系统

| 轮次 | 评审内容 | 审计领域特定检查项 |
|------|---------|-------------------|
| 第1轮 | 内容准确性审查 | 法规引用准确性、概念定义准确性、数据支撑准确性、事实陈述准确性 |
| 第2轮 | 逻辑连贯性审查 | 章节逻辑连贯性、概念层次连贯性、论证逻辑性 |
| 第3轮 | 适用性审查 | 审计指引实用性（P0级）、框架实用性、案例适用性 |
| 第4轮 | 格式规范化审查 | 标题层级规范、法规引用格式、术语使用一致性 |

### 2. 双模式优化

#### 修订模式（保守优化）
- **保留原稿核心观点和内涵**（严禁改变）
- 优化表述方式和逻辑结构
- 补充缺失信息，删除重复内容
- **字数控制在 95%-120%**
- 不需要用户确认观点和结构

#### 重构模式（大幅调整）
- 必须先提取并确认学术观点
- 必须先提出并确认新结构
- 可能大幅增删内容和调整结构
- 字数灵活但需说明理由
- 用户确认后才能开始重构

### 3. 多文件留痕输出（5个评审文件 + 1个优化稿）

#### 评审记录（1个）

## ⚠️ 核心原则（必须遵守）

### 1. 数据验证原则（最高优先级）

**绝对禁止编造任何内容、数据或引用材料**

#### 法规引用要求
- 必须使用法律法规库查询（Grep工具）
- 引用格式必须完整
- 禁止编造或猜测法律条款

#### 统计数据要求
- 必须使用联网工具查询并标注来源
- 必须标注发布机构和发布时间
- 如无法查询到准确数据，标注"[待核实]"

#### 数据验证优先级
```
优先级1（最高）：法律法规库 → 使用Grep在法规库中查询
优先级2（中等）：联网查询 → 使用WebSearch查询权威网站
优先级3（保底）：标注待核实 → 明确标注需要人工核对
```

### 2. 字数控制原则（严格标准）

**修订模式**: 优化稿字符数必须控制在原稿的 **95%-120%** 区间内

```
原稿字符数 = X
优化稿字符数 = Y
比例 = Y / X × 100%

合格标准：95% ≤ 比例 ≤ 120%
```

### 3. 模式选择原则

- **修订模式**: 适用于原稿有明确框架和核心观点
- **重构模式**: 适用于原稿结构混乱或观点不明确

## 🎓 审计领域分类

### 1. 财务审计 (Financial Audit)
- 核心任务：财务报表审计、披露审计、舞弊审计等
- 关键准则：CAS No. 1101, 1141, 1142, 1211, 1221, 1231, 1301, 1501
- 典型案例：上市公司财务造假、审计失败、重大错报审计

### 2. 内部控制审计 (Internal Control Audit)
- 核心任务：内控体系评价、控制缺陷识别、重大缺陷认定
- 关键框架：COSO框架、中国内控规范体系
- 典型案例：上市公司内控缺陷、重大内控失败

### 3. 文化遗产审计 (Cultural Heritage Audit)
- 核心任务：文化遗产保护审计、非遗保护审计、博物馆审计
- 关键法规：文物保护法、非遗法、保护条例
- 典型案例：文化遗产保护审计案例

### 4. 内部审计 (Internal Audit)
- 核心任务：运营审计、合规审计、风险管理审计
- 关键准则：IIA审计准则
- 典型案例：内部审计案例

### 5. 合规审计 (Compliance Audit)
- 核心任务：法律法规合规审计、行业监管合规
- 关键法规：行业监管规定
- 典型案例：合规审计案例
