# 审计书籍写作专业版 Skill

一个完整的审计书籍写作与评审工作流程，集成100个法律法规知识库，支持多领域、双模式、多文件留痕的审计书籍质量提升系统。

## 核心原则（必须遵守）

### ⚠️ 数据验证原则（最高优先级）

**绝对禁止编造任何内容、数据或引用材料**

#### 法律法规引用的严格要求

1. **必须使用法律法规库查询**
   ```bash
   # 使用 Grep 在法规库中查询（skill 内置）
   Grep -pattern "关键词" -path="references/laws-database/*.md"
   ```

2. **引用格式必须完整**
   ```markdown
   **《[法规全称]》**（[年份] [修正/修订/通过]）第[条款号]条
   ```

3. **禁止编造或猜测法律条款**
   - ❌ 禁止引用不确定或未经核实的法规
   - ❌ 禁止凭记忆或推测引用条款内容
   - ✅ 必须查阅法规原文

#### 统计数据引用的严格要求

1. **必须使用联网工具查询并标注来源**
   ```bash
   WebSearch "统计关键词 年份 site:权威网站"
   ```

2. **必须标注发布机构和发布时间**
   ```markdown
   [数据发布机构]. [年份]. [统计报告名称]. [发布时间].
   [来源：[机构官方网站URL]]
   ```

3. **如无法查询到准确数据**
   ```markdown
   **[待核实：该数据需要人工核对后补充]**
   ```

#### 案例数据引用的严格要求

1. **案例事实必须准确**
   - 公司名称、日期、金额等必须准确
   - 来源于公开审计报告或监管文件
   - 禁止编造案例或虚构事实

2. **法规依据必须准确**
   - 使用法规库查询验证
   - 标注完整的法规名称、条款号
   - 禁止引用不确定的条款

#### 数据验证优先级

```
优先级1（最高）：法律法规库 → 使用Grep在法规库中查询
优先级2（中等）：联网查询 → 使用WebSearch查询权威网站
优先级3（保底）：标注待核实 → 明确标注需要人工核对
```

#### 违规后果

- 发现编造数据或引用 → 整章节重新审核和修正
- 引用不完整 → 标注"[待核实]"并补充
- 法规条款错误 → 必须重新核实并修正

---

### ⚠️ 字数控制原则（严格标准）

#### 修订模式字数要求

**核心要求：** 优化稿字符数必须控制在原稿的 **95%-120%** 区间内

**计算公式：**
```
原稿字符数 = X
优化稿字符数 = Y
比例 = Y / X × 100%

合格标准：95% ≤ 比例 ≤ 120%
```

**检查时点：**
1. 生成优化稿时：预估字符数
2. 生成优化稿后：统计实际字符数

**超标处理：**
- 比例 < 95%：内容过度精简，**必须补充**缺失的重要内容
- 比例 > 120%：内容过度扩充，**必须精简**冗余表述
- 95%-120%：合格，符合优化要求

**统计命令：**
```bash
# 统计字符数（含空格）
wc -m <文件路径>
```

#### 重构模式字数要求

**核心要求：** 字数灵活，但必须说明变化理由

**说明要求：**
- 如字数增加 > 150%，说明新增了哪些重要内容
- 如字数减少 < 80%，说明删除了哪些冗余内容
- 在完成报告中详细说明字数变化原因

---

### ⚠️ 模式选择原则

**修订模式：** 适用于原稿有明确框架和核心观点
- 保留原稿核心观点和内涵（**严禁改变**）
- 优化表述方式和逻辑结构
- 补充缺失信息，删除重复内容
- 字数控制在 95%-120%
- 不需要用户确认观点和结构（除非发现重大问题）

**重构模式：** 适用于原稿结构混乱或观点不明确
- 必须先提取并确认学术观点
- 必须先提出并确认新结构
- 可能大幅增删内容和调整结构
- 字数灵活但需说明理由
- 用户确认后才能开始重构

## 四轮评审标准

### 第1轮：内容准确性审查

**审计领域特定检查项：**

1. **法规引用准确性**
   - 法规名称是否准确
   - 条款号是否正确
   - 发布日期、生效日期是否完整
   - 引用格式是否规范

2. **概念定义准确性**
   - 审计术语是否准确
   - 文化遗产概念是否准确
   - 专业术语使用是否统一

3. **数据支撑准确性**
   - 统计数据是否有来源
   - 案例数据是否准确
   - 资金数额是否准确

4. **事实陈述准确性**
   - 审计案例事实是否准确
   - 审计程序描述是否准确
   - 审计结论是否合理

### 第2轮：逻辑连贯性审查

**审计领域特定检查项：**

1. **章节逻辑连贯性**
   - 审计逻辑是否清晰
   - 程序描述是否连贯
   - 结论推导是否合理

2. **概念层次连贯性**
   - 概念体系是否完整
   - 概念关系是否清晰

3. **审计论证逻辑性**
   - 审计证据是否充分
   - 论证链条是否完整

### 第3轮：适用性审查

**审计领域特定检查项：**

1. **审计指引实用性**（P0级，800-1500字）
   - 范畴界定标准
   - 审计重点内容
   - 审计方法步骤
   - 常见问题处理

2. **框架实用性**
   - 概念框架表格
   - 分类对照表格
   - 标准对比表格
   - 问题-法规对照表

3. **案例适用性**
   - 典型案例说明
   - 审计过程描述
   - 审计发现总结
   - **审计启示**（800-1500字）

### 第4轮：格式规范化审查

**审计领域特定检查项：**

1. **标题层级规范**
   - 一级标题：第X章
   - 二级标题：一、二、三
   - 三级标题：（一）（二）（三）
   - 四级标题：1. 2. 3.

2. **法规引用格式**
   - 法规全称 + 年份 + 条款号
   - 统一格式，完整引用

3. **术语使用一致性**
   - 审计术语统一
   - 文化遗产术语统一

详见 `references/audit-book-templates.md`

4. 执行第3轮评审：适用性 → 生成评审记录
5. 执行第4轮评审：格式规范化 → 生成评审记录

8. 根据综合优化说明 → 生成优化稿

1. 读取原稿
2. 提取学术观点 → 生成学术观点确认文档
3. 等待用户确认
4. 提出结构优化方案 → 生成结构优化方案文档
5. 等待用户确认
6. 执行第1轮评审：内容准确性 → 生成评审记录
7. 执行第2轮评审：逻辑连贯性 → 生成评审记录
8. 执行第3轮评审：适用性 → 生成评审记录
9. 执行第4轮评审：格式规范化 → 生成评审记录
10. 汇总四轮评审 → 生成综合优化说明
11. 选择典型优化案例 → 生成优化对比
12. 根据综合优化说明 → 生成优化稿（重构版）
13. 统计成果和提升 → 生成完成报告
