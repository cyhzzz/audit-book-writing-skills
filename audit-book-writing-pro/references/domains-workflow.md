# 审计领域分类与工作流程

本文档整合了审计书籍写作所需的领域分类、数据验证标准和工作流程。

## 一、审计领域分类

### 1. 财务审计 (Financial Audit)

#### 审计方法
1. 风险导向审计 (Risk-Based Audit)
2. 循环审计法 (Cycle Approach)
3. 数据分析审计 (Data Analytics)
4. 持续审计 (Continuous Audit)
5. 计算机辅助审计技术 (CAATs)

#### 核心任务
- 财务报表审计 (Financial Statement Audit)
- 披露审计 (Disclosure Audit)
- 舞弊审计 (Fraud Audit)
- 关联交易审计 (Related Party Transaction Audit)
- 期初余额审计 (Opening Balance Audit)
- 期后事项审计 (Subsequent Event Audit)
- 持续经营审计 (Going Concern Audit)

#### 关键资料
**审计准则：**
- CAS No. 1101: 注册会计师的总体目标和审计工作的基本要求

- CAS No. 1142: 法律法规和注册会计师的责任
- CAS No. 1211: 通过了解被审计单位及其环境识别和评估重大错报风险
- CAS No. 1221: 计划和执行审计工作时的重要性
- CAS No. 1231: 针对评估的重大错报风险采取的应对措施
- CAS No. 1301: 审计证据
- CAS No. 1501: 审计报告

**监管规定：**
- 证券法及信息披露要求
- 上市公司信息披露管理办法
- 公开发行证券的公司信息披露内容与格式准则

**典型案例类型：**
- 上市公司财务造假案例
- 审计失败案例分析
- 重大错报审计案例
- 持续经营审计案例

#### 关键数据源
- 上市公司年度报告、招股说明书
- 证监会行政处罚决定书
- 交易所公开谴责、通报批评

## 二、数据验证详细指南

### 1. 数据验证原则

#### 1.1 核心原则

**⚠️ 绝对禁止编造数据或引用材料**

在审计书籍撰写中，任何数据、法规引用、案例材料都必须：

1. **来源于可验证的权威来源**
2. **经过查询工具核实**
3. **准确标注来源信息**
4. **无法核实的数据必须明确标注待核实**

#### 1.2 数据类型分类

| 数据类型 | 验证要求 | 查询工具 | 标注要求 |
|---------|---------|---------|---------|
| **法律法规** | 必须精准匹配 | Grep（法规库） | 法规名称、条款号、发布日期、生效日期 |
| **统计数据** | 必须有来源 | WebSearch/WebFetch | 发布机构、发布时间、来源网址 |
| **案例数据** | 必须准确 | 公开审计报告 | 公司名称、审计期间、数据来源 |
| **标准准则** | 必须查阅原文 | Grep（标准库） | 标准号、条款号、适用范围 |
| **学术观点** | 必须引用原文 | 学术论文/专著 | 作者、年份、期刊/出版社、页码 |

---

### 2. 法律法规引用规范（详细）

#### 2.1 必须使用法规库查询

**查询流程：**

```bash
# 步骤1：使用Grep在法规库中搜索关键词
Grep -pattern "文物保护法" -path="references/laws-database/*.md" -output_mode="content"

# 步骤2：找到相关法规后，读取完整内容
Read: references/laws-database/《中华人民共和国文物保护法》.md

# 步骤3：核实条款号和内容
Grep -pattern "第十条" -path="references/laws-database/《中华人民共和国文物保护法》.md" -output_mode="content" -C="2"
```

#### 2.2 引用格式要求

**标准格式：**

```markdown
**《[法规全称]》**（[年份] [修正/修订/通过]）第[条款号]条
```

**示例：**

```markdown
# 单一条款引用
根据**《中华人民共和国文物保护法》**（2024修正）第十条规定，
"国家发展文物保护事业，建立健全文物保护工作协调机制。"

# 多条款引用
**《中华人民共和国审计法》**（2021修正）第二条、第三条规定：
- 第二条：国家实行审计监督制度...
- 第三条：审计机关对...进行审计监督

# 条款内容完整引用
**《历史文化名城名镇名村保护条例》**（2008）第三条要求：
"历史文化名城、名镇、名村的保护应当遵循科学规划、严格保护的原则。"
```

#### 2.3 法规引用检查清单

- [ ] 法规名称准确无误
- [ ] 条款号正确
- [ ] 发布日期或修正年份标注
- [ ] 条款内容与原文一致
- [ ] 引用格式符合规范

### 3. 统计数据引用规范

#### 3.1 数据查询流程

**使用 WebSearch 查询：**

```bash
# 查询统计数据
WebSearch "全国审计机关 2023 年度 统计"

# 查询特定来源数据
WebSearch "文物保护单位 数量 2023 site:ncha.gov.cn"
```

#### 3.2 数据来源标注

**标准格式：**

```markdown
[发布机构]. [年份]. [统计报告名称]. [发布时间].
[来源：[官方网站URL]]
```

**示例：**

```markdown
国家文物局. 2023. 《2023年度全国文物资源概况》. 2024年1月.
[来源：国家文物局官方网站 www.ncha.gov.cn]

国家统计局. 2023. 《中国统计年鉴2023》. 2023年9月.
[来源：国家统计局网站 www.stats.gov.cn]
```

#### 3.3 无法核实的数据

**标注格式：**

```markdown
**[待核实：该数据需要人工核对后补充准确的统计来源]**
```

### 4. 案例数据引用规范

#### 4.1 案例来源要求

- 来源于公开审计报告
- 来源于监管处罚文件
- 来源于司法判决书
- 来源于官方媒体报道

#### 4.2 案例要素要求

**必需要素：**

| 要素 | 说明 | 示例 |
|------|------|------|
| 案例名称 | 准确的案例标题 | XX公司2023年度财务报表审计案例 |
| 审计对象 | 被审计单位名称 | XX股份有限公司 |
| 审计期间 | 审计时间范围 | 2023年1月1日至2023年12月31日 |
| 审计类型 | 审计类别 | 财务报表审计 |
| 审计机构 | 审计实施单位 | XX会计师事务所 |
| 关键数据 | 金额、数量等 | 涉及金额XXX万元 |
| 法规依据 | 违反的法规 | 违反《会计法》第X条 |
| 审计发现 | 具体问题 | 虚增收入XXX万元 |
| 审计建议 | 改进建议 | 建议完善XX内部控制 |

#### 4.3 案例引用示例

```markdown

2. 加强财务人员培训
```

## 四、常见问题处理

### Q1: 如何查询法规？
**A:** 使用 Grep 工具在 skill 内置的法规库中查询
```bash
Grep -pattern "关键词" -path="references/laws-database/*.md"
```

### Q2: 如何引用法规？
**A:** 使用标准格式
```markdown
**《[法规全称]》**（[年份] [修正/修订/通过]）第[条款号]条
```

### Q3: 审计应用指引要写多少字？
**A:** 800-1500字，包括范畴界定、审计重点、方法步骤、问题处理

### Q4: 案例启示要写多少字？
**A:** 800-1500字，从理论价值、实践价值、方法借鉴、风险警示四方面阐述

### Q5: 字数超标怎么办？
**A:**
- < 95%：补充缺失的重要内容
- > 120%：删除冗余表述，精简重复内容

### Q6: 如何选择审计领域？
**A:** 根据书籍目标读者和写作目的选择：
- 文化遗产审计 → 本 skill 重点
- 财务审计 → 会计师事务所
- 内部控制审计 → 企业内审
- 合规审计 → 监管合规
- 绩效审计 → 政府审计
- IT审计 → 信息技术审计
